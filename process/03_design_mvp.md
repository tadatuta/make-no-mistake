# Шаг 3: Проектирование MVP (UI/UX)
**Роль**: Designer
**Проект**: RepurposeAI (Рабочее название)

## 1. Концепция Дизайна
Поскольку мы целимся в инди-хакеров, SMM-специалистов и криэйторов, дизайн должен быть:
* **Минималистичным и "чистым"**: Много "воздуха" (whitespace).
* **Технологичным**: Темная тема (Dark Mode) по умолчанию, акценты неонового фиолетового или синего (цвета AI).
* **Сфокусированным на задаче**: Никаких лишних элементов, только инпуты и аутпут.
* **Стек стилей**: TailwindCSS (удобно и быстро).

## 2. Карта Экрана (Single Page Application)

### Header (Шапка)
* Логотип (текстовый: ✨ RepurposeAI).
* Кнопка "How it works?" (открывает модалку с инструкцией, как получить ключик Gemini).
* Ссылка на GitHub/X создателя.

### Секция навигации / настройки
* Поле ввода **Gemini API Key** (с кнопкой Save/Edit). Рядом иконка замочка и микро-текст: *"Your key is stored locally in your browser and never sent to our servers."*

### Основной Workspace (Разделен на две колонки на десктопе, друг под другом на мобилках)

#### Левая колонка (Ввод)
* **TextArea**: Большое поле "Paste your article, video transcript, or thoughts here..."
* **Конфигурация (Опционально)**: 
  * Tone of voice: "Professional, Casual, Viral, Educational".
* **Кнопка Вдува**: Огромная, привлекательная кнопка `[ ✨ Magic Repurpose ]`.

#### Правая колонка (Вывод / Результат)
* Табы (Вкладки) или Карточки друг под другом:
  1. **🐦 Twitter/X Thread**: Текст с разбиением на твиты (визуально как треды).
  2. **💼 LinkedIn Post**: Форматированный под LinkedIn текст (с отступами и эмодзи).
  3. **📝 SEO Blog Outline**: Структура статьи для блога.
* У каждого блока: кнопка "Copy" и "Regenerate".

## 3. Требования к разработке (Передача Developer'у)
* **Фреймворк**: React + Vite (быстро, легко деплоить на Vercel).
* **Стилизация**: TailwindCSS.
* **Иконки**: Lucide React.
* **Работа с API**: Прямые `fetch` запросы к `generativelanguage.googleapis.com` (Gemini API) из браузера.
* **Хранение данных**: `localStorage` для ключа API и последнего текста.

## 4. Рефлексия (Выводы из третьего шага)
* **Инсайт**: Важнейшая проблема конверсии в BYOK-модели (Bring Your Own Key) — страх за безопасность ключа. Поэтому UI должен кричать "Это безопасно, код исполняется локально!". Идеально даже дать ссылку на source code (Open Source), чтобы доказать прозрачность. 
* **Исправление плана**: Мы делаем проект Open Source, хостим на Vercel, добавляем в Header ссылку "View Source code on GitHub", чтобы повысить траст.
